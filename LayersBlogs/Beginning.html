so so much

The first problem was getting it to work, the one thing that makes me want to stop programming and change careers to beekeeping are linker and version incompatibility errors. Trying to follow the SDL's tutorial led to more headaches than progress, but it finally got working once I found "Lazy Foo' Productions"s excellent, detailed setup instructions. (Link)

Welp, thats about it for all the problems, now its just a matter of learning the SDL API and messing around with it to see what it can do, so here's what I found:

Its super basic, and is really similar to the java-based "Processing" framework that I've used before in undergrad graphics and when tutoring first years.
A general code structure could look like this:
  • init(): creates the window and other initilization parameters
  • keyHandler(): some form of handling key input
  • update(): a pure logic function for updating values per frame
  • draw(): a function that draws something onscreen
  • main(): that uses all the above functions that you can ignore after setting up
This general structure is robust, consistent, and most importantly can be broken up into more and more chunks that show the power of OOP in action.

Lets talk game dev:

Layers? Like onions and ogres? Yep, those.

Imagine you're playing mario 1-1, and as you're running to the right you can see 2-1 and 3-1 scrolling with you in the background. Now imagine you press a button (idk, press "E") and suddenly the "Layer" of 1-1 moves into the foreground and you start playing on 2-1, press E again and you're on 3-1. I feel like this type of world exploration would lend really well to a metroidvania or a puzzler, it could also be flipped on its head and the progression would be to go through the layers, the potential!

Or someone else already made a game with a similar theme but i don't know of any so I'm gonna pretend that I'm the first to come up with this idea.
